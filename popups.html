The same styling and layout is used for both modal popups and window popups, as seen in the example to the left.

The standard popup consists of three areas - a header, content area, and footer. The header contains the title of the popup. The content area contains the main content for the given popup. The footer contains any primary or secondary buttons associated with the popup. Each of the three areas has a 20px padding.

Modal Popup
When we want the user to complete an action one at a time, we use a modal popup. This ensures that the user cannot use the main interface while the modal popup is in view. Actions that require a modal popup include creating a custom search, creating a custom view, and bulk updating tasks.

Sample modal popup:


Window Popup
When we want to allow the user to perform multiple actions simultaneously, we use a window popup. As opposed to the modal popup, the user is allowed to have multiple windows open at a time. Actions that require a window popup include viewing the task audit history and viewing job-related notes.
